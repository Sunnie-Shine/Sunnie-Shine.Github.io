简介

六宫的对角线数独题目下存在两大潜规则。潜规则不同于技巧层面，它需要通过大量的题目证明得到结论，而其原理不同于技巧，原理不通过逻辑层面推理得到，而大多通过全部的答案盘枚举得到。

六宫对角线数独一共存在 12 种不等价盘面，如图所示。

它具有如下两个潜规则。

潜规则一：中心对称潜规则

中心对称潜规则指的是：盘面下关于盘面中心，中心对称的两格，填数一定不同。

例如这个图上给予了一个六宫对角线数独的题目盘面的答案，答案体现出，r1c2和r6c5的填数一定不同（实际上是一个2和一个3，确实是不同的）。其他的同理。

本潜规则可以直接在题目上使用，对于任意一个唯一解的六宫对角线数独题均适用。

潜规则二：轴对称潜规则

轴对称潜规则指的是：盘面上任取的四格，只要满足以下几个条件，则有结论：四格的填数至少有一个数会出现两次（四格填数不能完全不同）：

  1. 四格都不在对角线上；
  2. 将四格只存在于一行、一列之中，且一行一列之中各两格；
  3. 同在一行的这两格关于盘面中轴（c34中间的格线）轴对称、且同在一列的这两格关于盘面中轴（r34中间的格线）轴对称；
  4. 四格必须存在于对角线两侧，且两侧各两格；
  5. 两侧的四格必须一一关于对角线对称。

什么意思呢？如图所示，以下四格就满足上述给予的所有要求：

比如r25c6和r6c25四格，都不在对角线上；分属一行一列（r6和c6），而且恰有两格；r25c6确实对于盘面的中轴线轴对称、r6c25也是如此；四格也恰存在于捺对角线两侧，两侧也恰好各两格；并且r2c6和r6c2关于捺对角线对称、r5c6和r6c5关于捺对角线对称。

可空值类型和 bool? 简介

我们知道，C# 里含有值类型和引用类型两种东西，而 Java 里，则你只能产生引用类型，而 int 等类型，也都是它们提供好的特殊的实例。那么，由于 Java 采用引用类型的机制，使得了 Java 的完整整数型数据 Integer 是可以为 null 值的。而 C# 的值类型并不允许这样，因为 C# 里的值类型提供出来是为了作为基本数据类型和基元类型而存在的。说白了就是，这种类型基本上就是一个数据整体，一般迫不得已不拆开它们的内部数据，例如一个点对 Point，显然横纵坐标被拆解后完全显得无意义了，所以 Point 被作为一种值类型来看待。但 Java 受到语法的制约，我们只能写成引用类型。

那么，在 C# 里，我们可以为其声明含有空值的值类型数据，这种类型提供的 null 可是真的 null 哦！这种类型叫做 Nullable<T>，即可空值类型，其中的 T 必须是一个值类型。那么，Nullable<bool> 便是里面特殊的一种类型（可空值类型都可以简写为类型名加问号，即此处可以简写为 bool?）。由于布尔型数据的特殊性，这个类型只有两个数值，而加上可空类型的空值 null，这种类型就有了三个值：true、false 和 null。既然有值，就肯定有计算公式和方式，下面我们来看看它们的结果。

C# 语法查漏补缺系列（2）：访问修饰符

By 向向

发表于 2019-11-07

你知道 C# 里一共拥有多少个关键字吗？说 4 个的，肯定是从 Java 那边过来的。访问修饰符一共有 6 个哦！我们都来看看，它们都怎么用。

public 关键字

public 关键字就好比万能的东西，它在任何时间任何地点都可以看到。就好比一个公司里的员工，public 的员工就可以被任何时间、任何地点被其他人挖走。它们是永远可以暴露出去的，所以我们完全可以随意看到它们。

internal 关键字

internal 关键字仅仅在同一个项目里的任何位置可以被访问到。依然拿员工举例，这个公司里的员工如果用 internal 修饰，那么这个员工就只能在整个公司里才能看到，其它公司的人是不可能发现这些人的。

Java 的 package-private 模式类似于 internal，但区别比较大。Java 在类名前面不声明任何访问修饰符的时候，这个类类型将被当作是在整个包里可以看到，而其它包都看不到它。这是安全的，但显得范围比较窄。因为 Java 的包类似于文件夹，而 C# 里，一个项目内是可以含有很多文件夹的，不论你在哪个文件夹下，按照规则，它们都是可以被访问的，但 Java 的访问修饰符的设计模型就显得范围非常小了：出了文件夹就看不到了，这使得我们“迫不得已”去使用 public 关键字，但显然这样做并不保证安全性。

private 关键字

private 关键字当且仅当整个类体和结构体里才能看到，只要出了这个类或结构，任何地方都看不到。考虑如下例子：

protected internal 关键字

下面来看两个不一样的修饰符。首先是 protected internal 关键字。这个关键字来修饰一个数据，表示这个数据要么处于继承链上且向下可以访问，要么是整个项目里可以访问。换句话说，这个关键字组合就完全等同于 protected 或 internal。

private protected 关键字

最后来看一个比较特殊的关键字：private protected 关键字。这个关键字表示 protected 且 internal，即私有化继承。当你的继承链不希望被外界所继承到，我们可以使用 private protected 关键字来修饰类，来保证这个类在整个项目里的继承链上可以被向下继承访问到。

可以看到，这个访问行为应该使用 protected internal 来表示，好像更好更贴切一些，但因为这个关键字出现在 C# 7 之后，而 protected internal 在最初就已经出现了，所以我们不得不使用“私有继承”的这个说法来表示这种继承行为，即私有（private）的继承（protected）。

成员的默认访问修饰符

最后说一下成员的默认访问修饰符，还记得一个类体或一个结构体里可以包含哪些成员吗？下面罗列它们，并给出它们的默认访问修饰符。在默认情况下，我们是不用书写这些访问修饰符的，即“不写 = 缺省 = 默认”。

成员类型 默认的访问修饰符
类（class） internal
结构（struct） internal
接口（interface） internal
委托（delegate） internal
字段（field） private
构造器（constructor） private，有个默认不写的无参构造器是 public 的
析构器（finalizer） private
属性（property） private
事件（event） private
方法（method） private
操作符重载（operator overloading） private
隐式转换（implicit operator） private
显式转换（explicit operator） private
嵌套类（nested class） private
嵌套结构（nested struct） private
嵌套委托（nested delegate） private
接口里的属性（property in interface） public
接口里的事件（event in interface） public
接口里的方法（method in interface） public

由此可见，只要写在类体和结构体里的成员，默认都是 private；而处于整个命名空间，或全局范围内的成员，默认都是程序集里可用，即 internal；而接口体里的任何成员，则默认都是 public。
