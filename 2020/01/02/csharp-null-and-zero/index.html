缘起

很多小伙伴学过 C 语言，也明白 C 语言里一个复杂的机制：NULL 常量和 0 的转换。在 C99 里给出了 NULL 常量的精确定义：

它的书写结果是 (void *)0，似乎是直接把 0 直接强制转换为一个可以指向任意普通变量的一个指针对象。也就是说，这个 0 的意义从数值变为了指向第 0 号内存。

显然，这个逻辑是特殊且具有里程碑式的意义的，但它也带来了很多不合适的缺点，不过这里我们不作讨论。

总结

本文其实很简短，但也带有非常多的内容。

首先，在 C 语言里的 NULL 常量和 C# 里是一样的。当我们判断一个对象指针是否为空值时，我们可以使用安全的 null 关键字，也可以使用不安全的 (void*)0 来表示，它们是一样的。

不过，C 语言里的 NULL 常量并不是语言提供的，而是头文件里的，所以你需要使用头文件 stddef.h 才有办法找到它；相反，C# 使用更频繁而简单，它不需要引入任何命名空间，而直接使用关键字即可。当然，因为 C 和 C++ 是两门不同的编程语言，但很多地方都具有相似之处，所以你会看到定义的代码并不是上面这么简单，而是这样的：

在 C++ 里具有符号 __cplusplus，所以能够查找到这个符号也就意味着你当前使用的是 C++ 这门语言在写代码。如果是 C++ 的话，NULL 的定义应当为 0（当然前提是你没有定义 NULL）；如果不是 C++ 而是 C 语言，则定义结果是 (void *)0 了。
