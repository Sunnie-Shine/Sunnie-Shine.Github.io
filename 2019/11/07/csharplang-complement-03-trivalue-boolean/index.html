可空值类型和 bool? 简介

我们知道，C# 里含有值类型和引用类型两种东西，而 Java 里，则你只能产生引用类型，而 int 等类型，也都是它们提供好的特殊的实例。那么，由于 Java 采用引用类型的机制，使得了 Java 的完整整数型数据 Integer 是可以为 null 值的。而 C# 的值类型并不允许这样，因为 C# 里的值类型提供出来是为了作为基本数据类型和基元类型而存在的。说白了就是，这种类型基本上就是一个数据整体，一般迫不得已不拆开它们的内部数据，例如一个点对 Point，显然横纵坐标被拆解后完全显得无意义了，所以 Point 被作为一种值类型来看待。但 Java 受到语法的制约，我们只能写成引用类型。

那么，在 C# 里，我们可以为其声明含有空值的值类型数据，这种类型提供的 null 可是真的 null 哦！这种类型叫做 Nullable<T>，即可空值类型，其中的 T 必须是一个值类型。那么，Nullable<bool> 便是里面特殊的一种类型（可空值类型都可以简写为类型名加问号，即此处可以简写为 bool?）。由于布尔型数据的特殊性，这个类型只有两个数值，而加上可空类型的空值 null，这种类型就有了三个值：true、false 和 null。既然有值，就肯定有计算公式和方式，下面我们来看看它们的结果。

bool? 类型计算表
